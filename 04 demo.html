 Chain-of-Thought Prompting
  • Example: Step 1: Identify compliance criteria. Step 2: Compare regulation. Step 3: Identify gaps.
  • Purpose: Enhances structured reasoning and allows the AI to break down complex problems.
  • Practical Application: Provide a complex problem (e.g., "Analyze the budget implications of a new policy across three departments").
  • Task: Participants design a chain-of-thought prompt to guide an AI through the analysis steps. They will run the prompt in a sandbox environment (e.g., using a Gemini or Claude demo) and evaluate the step-by-step output.
Unit 2: Ethical and Responsible AI Use (2–3 Hours)
  1. Google Gemini (for general generative tasks, and for evaluating content for ethical concerns)
  2. OpenAI (ChatGPT) (for general generative tasks, and for evaluating content for ethical concerns)
  3. Anthropic (Claude) (for general generative tasks, and for evaluating content for ethical concerns)
  4. Llama 2 / Llama 3 (as examples of open-weights models for sovereign control and ethical fine-tuning)
  5. Hugging Face (as an ecosystem for exploring and evaluating open-source models, potentially for bias)
  6. GitHub Copilot / Replit AI (as examples of code-generating AIs where governance and audit trails are critical)
  7. Midjourney / DALL-E (as examples of image-generating AIs where ethical considerations like bias in representation are relevant)
  8. Internal Knowledge Management AI Systems (conceptual, for synthesis and information retrieval within secure government environments)
  9. Enterprise AI Governance Platforms (conceptual, for managing AI models, tracking usage, and ensuring compliance, e.g., features found in Azure AI Governance or Google Cloud AI Platform Unified)
  10. Data Logging Tools (conceptual, integrated into AI systems for audit trails)
  11. Privacy-Enhancing AI Techniques / Data Anonymization Tools (conceptual, e.g., differential privacy libraries, synthetic data generation for sensitive data handling)
  12. AI Bias Detection Toolkits (conceptual, e.g., functionalities similar to IBM AI Fairness 360 or Google's What-if Tool)
  13. XAI (Explainable AI) Frameworks (conceptual, for understanding "why" AI made a decision, often integrated into specialized AI models)
 Human-in-the-Loop Principles
  • Concept: Emphasizing AI as an assistant, not an authority. Human decision-making and responsibility remain paramount.
  • Frameworks: Introduction to output validation frameworks (e.g., cross-referencing, logical consistency checks) and escalation logic (when to bypass AI).
  • Interactive Scenario: Participants review a brief AI-generated summary of citizen feedback for a new public service. They must:
    1. Identify one potential factual error or missing critical context that a human would immediately spot.
    2. Describe the specific human validation steps they would take to rectify this, referencing the "assistant, not authority" principle.
    3. Reflect on a situation where they would bypass AI for this task entirely, and why.
  • Tool Reference: Applicable to all generative AI tools like Google Gemini, OpenAI (ChatGPT), Anthropic (Claude), and internal knowledge management AI systems used for synthesis.
Unit 3: AI Content Generation and Media Creation (3–4 Hours)
  1. Google Gemini (General generative text, summarization, conceptual image generation, multimodal capabilities)
  2. OpenAI (ChatGPT) (General generative text, summarization, creative writing)
  3. Anthropic (Claude) (General generative text, particularly for longer documents and nuanced understanding, ethical reasoning)
  4. Llama 2 / Llama 3 (Examples of open-weights models for custom text generation, potentially fine-tuned for specific government communication styles, image generation capabilities)
  5. Hugging Face (Ecosystem for exploring and deploying various open-source models across modalities, including text, image, and audio)
  6. DALL-E (Image generation)
  7. Midjourney (Image generation, often with more artistic control and distinct styles)
  8. Stable Diffusion (Open-source image generation, highly customizable for specific visual outputs)
  9. Adobe Firefly (Conceptual, for integrated generative AI within creative suites, especially for image editing, generation, and potentially graphic design)
  10. ElevenLabs (High-quality AI voice generation, voice cloning)
  11. Google Text-to-Speech (API for generating synthetic speech for various applications)
  12. Synthesia (AI video generation from text, including avatars and customizable scenes)
  13. Descript (AI-powered video editing, transcription, and voice cloning for audio/video summaries, podcast editing)
  14. RunwayML (Conceptual, for advanced generative video, AI-powered video editing features, and cinematic effects)
 Interactive Exercise
  • Task: Participants are given a complex, unstructured set of data and a specific government communication objective (e.g., "Draft a new internal policy directive on remote work procedures, including sections for eligibility, responsibilities, and approval process").
  • Process: They must use a range of AI sandbox’ (e.g., Google Gemini or ChatGPT demo) to generate a policy draft.
  • Analysis: They will then critically analyze the AI's output for structural integrity, tone, and compliance, identifying areas where human legal or policy expertise is irreplaceable.
  • Tool Reference: Advanced LLMs like Google Gemini, OpenAI (ChatGPT), Anthropic (Claude), and conceptual specialized legal/policy drafting AIs (e.g., those trained on vast legal corpora) if available.
Unit 4: AI Agents and Workflow Automation (12 Hours)
  1. OpenAI Custom GPTs (Primary tool for agent building)
  2. OpenAI Actions API (For external tool invocation by GPTs)
  3. n8n (Primary visual automation platform for workflow design)
  4. Google Gemini (As a core LLM for agentic reasoning, also for its multimodal capabilities)
  5. OpenAI (GPT-4 and other models) (As core LLMs for agentic reasoning)
  6. Anthropic (Claude) (As a core LLM for agentic reasoning, particularly for complex text processing)
  7. Llama 2 / Llama 3 (Examples of open-weights LLMs that could power custom agents in a sovereign environment)
  8. Hugging Face (Ecosystem for exploring and deploying various open-source models that could serve as agent components or tools)
  9. Zapier / Make (formerly Integromat) (Conceptual, as alternative no-code/low-code automation platforms)
  10. Microsoft Power Automate (Conceptual, for enterprise-level automation and integration within Microsoft ecosystems)
 OpenAI Agent Builder & Advanced Configuration (3 hours 45 minutes)
  • Custom GPTs: Creation & Actions API
  • Concept: Hands-on introduction to creating Custom GPTs (or similar agent-building features in other platforms).
  • Config: Configuring the agent's instructions, capabilities, and integrating custom "Actions" via OpenAPI specifications to allow interaction with external systems.
  • Walkthrough: Step-by-step demonstration of creating a Custom GPT, defining its purpose, and setting up a simple mock Action (e.g. a "lookup" API call that returns static data) within the OpenAI environment.
  • Tool Reference: OpenAI Custom GPTs, OpenAPI Specification (conceptual for Actions API).
Unit 5: Vibe Coding – Build AI-Powered Micro Tools (4 Hours)
  1. GitHub Copilot (Primary AI code assistant for generation, debugging, refactoring)
  2. Google Gemini Code (AI code assistant capabilities, integrated with Google's ecosystem)
  3. Replit AI (Online IDE with integrated AI coding assistance)
  4. Code Llama (Conceptual, as an open-source large language model specifically for code generation and understanding)
  5. OpenAI (ChatGPT/GPT-4) (General generative LLMs used for code generation from natural language, debugging explanations, and refactoring suggestions)
  6. Anthropic (Claude) (General generative LLMs used for code generation, debugging, and secure code review discussions)
  7. Streamlit (Python library for quickly building simple web apps/dashboards with minimal code)
  8. Flask (Python web framework for building simple web applications)
  9. n8n (Visual automation tool, relevant for building "Mini workflow tools" with AI integration, as covered in Unit 4)
  10. Python (basic scripting environment) (Underlying language for many micro-tools)
  11. HTML/CSS/JavaScript (basic web technologies) (For building simple web-based interfaces)
  12. Internal Knowledge Management AI Systems (Conceptual, for retrieving documentation relevant to coding tasks)
  13. Enterprise AI Governance Platforms (Conceptual, for managing and monitoring internally developed micro-tools)
  14. Data Logging Tools (Conceptual, for maintaining audit trails of micro-app usage and data processing)
  15. Privacy-Enhancing AI Techniques / Data Anonymization Tools (Conceptual, relevant when micro-apps handle sensitive data)
  16. AI Security Scanners (Conceptual, for identifying vulnerabilities in AI-generated code, e.g., features in Snyk Code AI or DeepCode AI)
  17. XAI (Explainable AI) Frameworks (Conceptual, for understanding decisions made by AI in code generation or debugging)
 AI-Assisted Development: Generate, Debug, Refactor
  • Code Generation: Using natural language prompts to generate simple code snippets (e.g., Python script for data processing, HTML for a basic web page).
  • Debugging: Leveraging AI to identify and explain errors in code, and suggest corrections.
  • Refactoring: Asking AI to optimize or improve existing simple code for clarity or efficiency.
  • Guided Mini-Practical: Participants will use an AI code assistant sandbox (e.g., GitHub Copilot or Google Gemini Code demo):
    1. Generate simple script.
    2. Debug (Simulated) provided pre-written error.
    3. Refactor for readability/comments.
  • Tool Reference: GitHub Copilot, Google Gemini Code, Replit AI, Code Llama.

 General Lab Requirements
  • Participants will utilize AI tools in a sandbox environment.
  • Submission includes:
    • Initial Prompt(s)
    • AI's Raw Output
    • Human-Refined Output
    • Justification of Edits/Design (75-100 words)
    • Key Human-in-the-Loop Point
